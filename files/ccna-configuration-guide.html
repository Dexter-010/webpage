1) Base Device Hardening & SSH Management Access

ðŸ”¹ 1. System Basics

hostname R1
no ip domain-lookup
ip domain-name lab.local
  • Hostname: Identifies the device (R1).
  • Disable DNS lookup: Prevents delays when mistyping commands.
  • Domain name: Required for SSH key generation (lab.local).

ðŸ”¹ 2. Passwords & Encryption

service password-encryption
enable secret <STRONG_SECRET>
username admin secret <STRONG_SECRET>
  • Password encryption: Scrambles plaintext passwords (weak, but better than nothing).
  • Enable secret: Secure, encrypted password for privileged EXEC mode.
  • Local user account: Creates admin with a secure encrypted password.

ðŸ”¹ 3. SSH Security

crypto key generate rsa modulus 2048
ip ssh version 2
ip ssh time-out 60
ip ssh authentication-retries 3
  • RSA keys: Needed for SSH encryption (2048 bits = strong).
  • Force SSH v2: Ensures only secure SSH sessions.
  • Timeout: 60 seconds to authenticate.
  • Retries: Maximum 3 login attempts.

ðŸ”¹ 4. Console Line Configuration

line console 0
 logging synchronous
 exec-timeout 10 0
 password <OPTIONAL>
 login local
  • Console: Configures local (physical) access.
  • Logging synchronous: Stops logs from interrupting typing.
  • Exec-timeout: Auto logout after 10 minutes.
  • Login local: Uses username admin instead of only a password.

ðŸ”¹ 5. VTY (Remote Access)

line vty 0 15
 transport input ssh
 exec-timeout 10 0
 logging synchronous
 login local
  • VTY lines: Configure remote access (0â€“15 = 16 sessions).
  • SSH only: Blocks Telnet.
  • Timeout: Auto logout after 10 minutes.
  • Local login: Requires username & password.

ðŸ”¹ 6. Banner

banner motd ^
  Authorized access only! ^
  • Message of the Day: Displays warning before login.
  • Acts as a legal security notice.

Verify: show ip ssh, show users, show run | section line vty

8) NAT (PAT/Overload, Static, Dynamic Pool)

PAT (most common for Internet edge)

conf t
access-list 1 permit 192.168.10.0 0.0.0.255
access-list 1 permit 192.168.20.0 0.0.0.255
!
interface g0/0          ! WAN/ISP
 ip address 203.0.113.2 255.255.255.248
 ip nat outside
interface g0/1          ! LAN
 ip address 192.168.10.254 255.255.255.0
 ip nat inside
!
ip nat inside source list 1 interface g0/0 overload
end

Static 1:1 NAT (publish an internal server)

conf t
ip nat inside source static 192.168.10.10 203.0.113.10
! Ensure inside/outside set on interfaces
end

Dynamic NAT Pool

conf t
ip nat pool NETPOOL 203.0.113.100 203.0.113.110 netmask 255.255.255.0
access-list 10 permit 192.168.10.0 0.0.0.255
ip nat inside source list 10 pool NETPOOL
end

Verify: show ip nat translations, show ip nat statistics

10) OSPFv2 (Single Area) for IPv4
conf t
router ospf 1
 router-id 1.1.1.1
 passive-interface default
 no passive-interface g0/0
 no passive-interface g0/1
 network 10.0.0.0 0.0.0.255 area 0
 default-information originate   ! if a default route exists
end
!
! Or interface-style
interface g0/0
 ip ospf 1 area 0
 ip ospf network point-to-point   ! if needed on p2p links

Useful tweaks: ip ospf cost 100 (on interface), neighbor not used in broadcast; for NBMA/p2p you may.

Verify: show ip protocols, show ip ospf neighbor, show ip ospf interface brief, show ip route ospf

20) Troubleshooting Playbook (Fast)
  1. Layer 1: show int status (errâ€‘disable? notconnect? speed/duplex?)
  1. VLAN: Is the access port on the right VLAN? show vlan brief
  1. Trunking: Allowed/native VLANs match? show int trunk
  1. STP: Blocked ports? show spanning-tree
  1. IP: Address/gateway correct? show ip int brief
  1. Routing: Does route exist? show ip route (look for C, S, O)
  1. ACL/NAT: Check for unexpected drops: show access-lists (hit counts), show ip nat stats
  1. HSRP: VIP reachable? Who is Active/Standby? show standby brief
  1. DNS/NTP: Time skew breaks certs/SSH; fix clock & NTP.
  1. Logs: show logging for clues.

22) Exam Gotchas & Best Practices
  • Trunk encapsulation command may be absent on some switches (dot1q only). If so, omit it.
  • Always match native VLAN on both sides; avoid using VLAN 1 for trunks/native.
  • For OSPF default-information originate, ensure a default route exists first.
  • Use interface-style OSPF to avoid wildcard math in timed exams.
  • On IPv6 statics with a linkâ€‘local next-hop, include the exit interface.
  • After enabling ip routing on L3 switches, donâ€™t set ip default-gateway; use ip route instead.
  • For port-security, donâ€™t forget to set access mode + access VLAN first.
  • For DHCP snooping/DAI, mark only uplinks as trusted.
  • Save often. Verify with show commands after each section.
